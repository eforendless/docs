## Structure

Each parameter in the embed is represented by a key-value pair which is separated by a colon. For example, the `title` parameter will look like `{title: hello {user}}`. The `{user}` is a [variable](/client/variables) which will be filled in with the user's display name.

- `{` begins a parameter.
- `:` separates parameter from content.
- `$v` separates the parameters.
- `}` ends a parameter.

### Parameters

- `url` - The embed URL. (`https://..`)
- `color` - The embed color. (`#FFFFFF`)
- `title` - The embed title. (`hello {user}`)
- `description` - The embed description. (`hello {user}`)
- `image` - The embed image URL. (`https://..`)
- `thumbnail` - The embed thumbnail URL. (`https://..`)
- `timestamp` - The embed timestamp. (**NO ARGUMENTS**)

The following parameters require additional arguments which are separated by `&&`.

### Variables

You can use dynamic variables in your embeds to display user-specific information. For example, `{user}` will be replaced with the user's display name.

<Info>You can view the available variables by clicking [here](/discord/variables).</Info>

```
{description: {guild.name}}$v{message: Hello! {user.mention}}
```

## Templates

Velveta Bot supports embed templates that can be saved and reused across your server. Templates make it easy to maintain consistent messaging and styling.

### Creating Templates

Define your template with embed parameters and store it in your staff docs or notes:

```
{title: Welcome to {guild.name}!}$v{description: Hello {user.mention}, thanks for joining!}$v{color: #00FF00}$v{thumbnail: {guild.icon}}
```

### Using Templates

Paste your template into the appropriate moderation or announcement flow for your server.

### Template Categories

Organize your templates with categories:
- `welcome` - Welcome messages
- `announcements` - Server announcements  
- `rules` - Server rules and guidelines
- `events` - Event notifications
- `moderation` - Moderation notices
- `custom` - Custom templates

## Advanced Features

### Conditional Content

Use conditional statements to show different content based on user properties:

```
{description: {if:{user.premium}:Welcome VIP member!:Welcome to the server!}}
```

### Dynamic Colors

Set embed colors based on user roles or server settings:

```
{color: {user.role.color}}
```

### Reactions and Buttons

Add interactive elements to your embeds:

```
{title: Choose your role}$v{button: green && Get Verified && verify}$v{button: blue && View Rules && rules}
```

## Frequently Asked Questions

### How can I add a new line to my description parameter?

If you want to add a new line to your description parameter, you can press `SHIFT` + `ENTER` to create a new line, or use `\n` in your embed code.

### Why isn't my embed code working for some commands?

Some commands allow both raw text and embed code. Because of this, you'll need to specify if you're entering an embed. To do this, you'll need to begin your embed code with `{embed}$v` then proceed to write your embed code after as normal.

```
/welcome add channel:#general message:{embed}$v{title: Welcome to {guild.name}}$v{description: Welcome {user.mention}}
```

### How do I make fields inline?

Add `inline` at the end of your field value:

```
{field: Server Name && {guild.name} && inline}
```

### Can I use custom images in embeds?

Yes! You can upload images to Velveta's CDN and use them in embeds:

```
{image: https://cdn.velvetaai.com/your-image.png}
```

### How do I test my embed before using it?

Post your embed in a staff-only test channel first to verify formatting.

### What's the character limit for embed fields?

- **Title**: 256 characters
- **Description**: 4096 characters
- **Field Name**: 256 characters
- **Field Value**: 1024 characters
- **Footer**: 2048 characters
- **Author Name**: 256 characters

### Can I save embed templates for later use?

Yes! Reuse snippets across your server by saving them in staff docs or using your server's preferred tooling.
