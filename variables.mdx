## Overview

Variables are dynamic placeholders that get replaced with actual values when your commands are executed. They allow you to create personalized and contextual responses that adapt to different users, servers, and situations.

## User Variables

Access information about the user who triggered the command.

<AccordionGroup>
    <Accordion title="Basic User Info">
        - `{user}` - User's display name
        - `{user.mention}` - Mention the user (@User)
        - `{user.tag}` - Username with discriminator (User#1234)
        - `{user.username}` - Username only
        - `{user.discriminator}` - Discriminator only (#1234)
        - `{user.id}` - User's Discord ID
        
        <CodeGroup>
            ```javascript Example
            Welcome {user.mention} to {guild.name}!
            ```
            
            ```javascript Output
            Welcome @JohnDoe to My Awesome Server!
            ```
        </CodeGroup>
    </Accordion>
    
    <Accordion title="User Avatar & Profile">
        - `{user.avatar}` - User's avatar URL
        - `{user.banner}` - User's banner URL
        - `{user.accent_color}` - User's accent color
        - `{user.created_at}` - Account creation date
        - `{user.joined_at}` - Server join date
        
        <CodeGroup>
            ```javascript Example
            {thumbnail: {user.avatar}}$v{description: {user} joined on {user.joined_at}}
            ```
        </CodeGroup>
    </Accordion>
    
    <Accordion title="User Status">
        - `{user.status}` - Online status (online, idle, dnd, offline)
        - `{user.activity}` - Current activity
        - `{user.premium}` - Premium status (true/false)
        - `{user.bot}` - Bot status (true/false)
        - `{user.verified}` - Verification status
        
        <CodeGroup>
            ```javascript Example
            {user} is currently {user.status}
            ```
        </CodeGroup>
    </Accordion>
</AccordionGroup>

## Conditional Variables

Create dynamic content based on conditions.

<AccordionGroup>
    <Accordion title="If Statements">
        Use conditional logic to show different content:
        
        ```javascript Syntax
        {if:condition:true_value:false_value}
        ```
        
        <CodeGroup>
            ```javascript Example
            {if:{user.premium}:Welcome VIP member!:Welcome to the server!}
            ```
            
            ```javascript Nested Conditions
            {if:{user.verified}:{if:{user.premium}:Verified VIP:Verified Member}:Unverified User}
            ```
        </CodeGroup>
    </Accordion>
    
    <Accordion title="Conditions">
        Available conditions for if statements:
        
        - `{user.premium}` - User has premium
        - `{user.verified}` - User is verified
        - `{user.bot}` - User is a bot
        - `{guild.premium}` - Server has premium
        - `{channel.nsfw}` - Channel is NSFW
        
        <CodeGroup>
            ```javascript Example
            {if:{guild.premium}:This server has premium features!:Upgrade to unlock premium!}
            ```
        </CodeGroup>
    </Accordion>
</AccordionGroup>

## Custom Variables

Create and use custom server-specific variables.

<AccordionGroup>
    <Accordion title="Server Variables">
        Set custom variables for your server:
        
        ```
        /variable set name:server_motto value:We are awesome!
        ```
        
        Then use in embeds:
        ```javascript
        {description: {custom:server_motto}}
        ```
    </Accordion>
    
    <Accordion title="User Variables">
        Set custom variables for users:
        
        ```
        /variable set user:@user name:nickname value:SuperGamer
        ```
        
        <CodeGroup>
            ```javascript Usage
            {if:{custom.user:nickname}:Hey {custom.user:nickname}!:Hello {user}!}
            ```
        </CodeGroup>
    </Accordion>
</AccordionGroup>

## Best Practices

### Performance Tips

1. **Minimize Complex Variables**: Use simple variables when possible
2. **Cache Custom Variables**: Avoid frequent custom variable lookups
3. **Test Before Deployment**: Always test complex variable combinations

### Security Considerations

1. **Sanitize User Input**: Custom variables are automatically sanitized
2. **Avoid Sensitive Data**: Don't store sensitive information in variables
3. **Validate Permissions**: Check user permissions before using certain variables

### Common Patterns

<CodeGroup>
    ```javascript Welcome Message
    {title: Welcome to {guild.name}!}$v{description: Hello {user.mention}, you're member #{guild.member_count}!}$v{thumbnail: {user.avatar}}
    ```
    
    ```javascript Moderation Log
    {title: Moderation Action}$v{description: {target.mention} was {case.type} by {case.moderator}}$v{field: Reason && {case.reason}}$v{timestamp}
    ```
    
    ```javascript Server Stats
    {title: {guild.name} Statistics}$v{field: Members && {guild.member_count:comma} && inline}$v{field: Online && {guild.online_count:comma} && inline}$v{field: Boost Level && {guild.boost_level} && inline}
    ```
</CodeGroup>
